# PD 部署决策冲突检测设计

## 背景

当前 PD 部署规划工具的四步决策流程：
1. 第 0 优先级：模型规模（≤8B 默认混部）
2. 第 1 优先级：场景（在线推理→分离，数据生产→混部）
3. 第 2 优先级：长度比（α>0.7→分离）
4. 第 3 优先级：SLO（TPS优先→混部，时延优先→分离）

当用户输入存在内在矛盾时（如"TPS优先"但"S_in >> S_out"），不同优先级可能给出不同结论，导致决策冲突。

## 目标

在决策过程中检测冲突，主动提示用户确认，避免基于矛盾输入给出误导性结论。

## 设计方案

### 插入位置

在**第二步（PD策略决策）**之后、**第三步（遍历并行策略）**之前，插入冲突检测模块。

### 冲突定义

当第二步中**两个或以上优先级给出不同 PD 策略结论**时，触发冲突检测。

### 冲突提示格式

```markdown
⚠️ 检测到决策冲突

## 各优先级决策结果

| 优先级 | 因素 | 输入值 | 推荐 |
|--------|------|--------|------|
| 第0优先级 | 模型规模 | [值] | PD混部/PD分离 |
| 第1优先级 | 场景 | [值] | PD混部/PD分离 |
| 第2优先级 | 长度比 | α=[值] | PD混部/PD分离 |
| 第3优先级 | SLO | [值] | PD混部/PD分离 |

## 请确认

1. **您的输入信息是否有误？** 如有误请直接输入修正值
2. **或确认坚持某种部署方式？** 直接说"PD混部"或"PD分离"
```

### 后续处理逻辑

| 用户反馈 | 系统行为 |
|----------|----------|
| 修正输入 | 重新走决策流程（第二步开始） |
| 确认部署方式 | 跳过冲突，用用户选择的方案继续第三步 |
| 无反馈 | 等待用户输入 |

### 无冲突时

直接进入第三步，不显示额外信息。

## 实现要点

1. 冲突检测是**被动触发**，只有检测到冲突才提示
2. 提示内容**简洁**，只展示各优先级结论和请求确认
3. **不替用户做选择**，最终决策权在用户
