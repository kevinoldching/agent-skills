## 2. PD 策略决策

### 决策结果
**推荐策略: PD混部**

### 决策依据

| 优先级 | 因素 | 分析 |
|--------|------|------|
| 场景 | chat | Chat场景通常为在线推理，但用户明确TPS优先，混部可最大化吞吐量 |
| 长度比 | α=0.40 | α=1024/(1024+3×512)=0.4，处于[0.3-0.7]基本平衡范围，混部或分离均可 |
| SLO | TPS优先 | TPS优先时，混部无KV Cache传输开销，所有卡都可用于decode，吞吐量更高 |

### α 计算公式
```
α = S_in / (S_in + k × S_out), k=3 (default)
α = 1024 / (1024 + 3 × 512) = 1024 / 2560 = 0.4
imbalance = max(α, 1-α) = 0.6
imbalance < 0.8 → 混部
```

## 4. 并行策略配置

### PD 混部部署（推荐）

**配置: TP=8, EP=8, DP=1（8卡混部）**

| 参数 | 值 |
|------|-----|
| 总GPU数 | 8 |
| Tensor Parallel | 8 |
| Expert Parallel | 8（MoE模型，128专家） |
| Data Parallel | 1 |
| 每卡显存占用 | ~9.67 GB (batch=1) |
| 最大Batch Size | ~900+ |
| 预估吞吐量 | 需根据实际TTFT/TPOT测定 |

### 备选配置

| 配置 | 总卡数 | TP | EP | DP | 最大Batch | 每卡权重(GB) |
|------|--------|----|----|----|-----------|-------------|
| TP=4, EP=4, DP=1 | 4 | 4 | 4 | 1 | ~768 | 15.21 |
| TP=8, EP=8, DP=1 | 8 | 8 | 8 | 1 | ~900 | 7.62 |

### EP 配置说明（MoE模型）

对于gpt-oss-120b MoE模型（128专家）：
- EP > 1 是必须的，因为需要将专家分布到多个设备
- EP=8 表示将128个专家分配到8个设备，每个设备处理16个专家
- EP=4 表示将128个专家分配到4个设备，每个设备处理32个专家
- 约束：EP <= min(总卡数, 专家数量) = min(8, 128) = 8

### 性能估算

```
混部 TPS 公式: TPS = batch_size × (S_in + S_out) / (TTFT + S_out × TPOT)

假设配置: TP=8, EP=8, DP=1, batch=512
- S_in = 1024, S_out = 512
- 总tokens/batch = 512 × 1536 = 786,432
- 若 TTFT ≈ 100ms, TPOT ≈ 10ms/token
- TPS ≈ 512 × 1536 / (100 + 512 × 10) = 786432 / 5220 ≈ 150K tokens/s
```

**注意**: 以上为估算值，实际TPS需根据Ascend 910B实际算力测定。

## 5. 实现注意事项

### 硬件配置
- Ascend 910B 单机通常为8卡配置
- 建议选择 **8卡混部配置 (TP=8, EP=8, DP=1)** 以获得最佳TPS

### MoE模型优化
1. **Expert负载均衡**: 确保128个专家在8个设备间均匀分布
2. **通信优化**: EP通信为All-to-All，需关注带宽利用
3. **量化支持**: 模型使用MXFP4量化，需确认部署框架支持

### 显存优化建议
1. **动态Batch**: 根据实际请求调整batch_size
2. **KV Cache优化**: 使用SWA (Sliding Window Attention) 减少KV Cache占用
3. **激活值优化**: 混部时激活值占用较高，可适当降低batch

### xPyD格式说明
本方案采用PD混部，无需xPyD格式。若后续需PD分离：
- y = 1 (固定)
- Prefill总卡数 = Decode总卡数
- x × TP_prefill × DP_prefill = TP_decode × DP_decode

## 6. 关键公式参考

| 公式 | 说明 |
|------|------|
| `TPS = batch_size × (S_in + S_out) / (TTFT + S_out × TPOT)` | 混部 TPS |
| `α = S_in / (S_in + k × S_out)` | 输入输出长度比 |
| `KV Cache = (18 × batch × seq × 1024 + 18 × min(batch × seq, 128) × 1024) / (TP × CP)` | KV Cache计算 |
| `EP = TP × DP` | EP与TP×DP关系 |
